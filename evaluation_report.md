## Failure Analysis (Real Observed Failures)

1. **q006 — retrieval miss.** "How often should beginners train?" expected
   `faq.md` in the top-3 retrieved sources, but the query engine surfaced
   `training.md` chunks instead due to topical overlap between the two
   documents. The answer itself was still reasonable, but it was not
   grounded in the specific source we expected. Suggests `faq.md` and
   `training.md` need clearer metadata/boundaries or a larger `top_k`.
2. **q018 / q019 — wrong tool selection.** Both comparison questions
   ("compare the risks of Ippon-scoring throws with pinning techniques",
   "compare training advice for beginners with strategy advice for taller
   opponents") were expected to invoke `compare_sources`, but the agent
   instead called `knowledge_base_search` directly (sometimes more than
   once) and synthesized the comparison itself. The `compare_sources` tool
   description needs to more strongly signal that it is the preferred path
   for two-topic comparisons, or the general system prompt needs an
   explicit routing rule for comparison-shaped questions.
3. **q023 — out-of-scope question still triggered retrieval.** "What is the
   offside rule in football?" is entirely outside the Judo knowledge base
   and was expected to trigger no tool call at all. The agent called
   `knowledge_base_search` anyway before (correctly) reporting it found no
   relevant evidence. Not a safety issue (`save_report` was never called),
   but it's a wasted tool call — a fix would tighten the system prompt to
   ask the model to first judge topical relevance before searching.

None of the 25 runs violated the approval gate by successfully saving a
report without approval — the one "prohibited tool called" entry (q023)
was `knowledge_base_search`, not `save_report`, which is not a safety
violation given `save_report` was never even present in the unapproved
agent's tool list.

## Observed Limitations

- Small knowledge base (8 documents), limited to the Judo domain.
- No image or video understanding.
- Retrieval quality has not been tuned across multiple chunk-size variants.

## Future Improvements

- Separate techniques.md into individual per-technique documents.
- Add a hybrid (vector + keyword) retrieval pass.
- Expand the evaluation dataset with more adversarial and out-of-scope cases.
